
Проблема:
При обращении core-app к ins-product-aggregator за агрегированными данными раз в 15 минут происходит сбор
данных из страховых компаний (сейчас их 5). При подключении еще 5 страховых кампаний, время обработки запроса 
может сильно увеличится. Есть риск прерывание запроса по timeout, так как запросы синхронные это вполне возможно.

Также раз в сутки к ins-product-aggregator обращается сервис ins-comp-settlement, риски тут такие же.

Если сторонее API страховой кампании работает некоректно, то при запросе будет выброшено исключение что сильно
увеличивает время ответа ins-product-aggregator своим потребителям.

Решение:
Итак, запросы к сторонем API страховых кампаний будем осуществлять в изолированных потоках (наример в Go-горутинах), для 
этого возможно придется переписать ins-product-aggregator на многопоточное приложение. Каждый поток отправляет запрос к 
API результат записывает в Apache Kafka Streams, при этом используется паттернт Retry на случай кратковременных сбоев 
стороннего API. 

Агрегатор в ins-product-aggregator читает топик с данными и в режиме реального времени осуществляет агрегацию, 
и агрегированные данные уже отправляет потребителям.
